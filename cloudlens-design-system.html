01

Identity & Brand Personality

CloudLens is a precision developer tool. Feels like a terminal with a conscience — fast, direct, technically credible. It never markets. It informs.

Precise
Every number is real. Every label is exact. No rounding up, no marketing superlatives in data surfaces.
Terse
Fewer words, more meaning. "3 AWS services" beats "We detected 3 Amazon Web Services integrations."
Useful
Every screen answers a question the developer is already asking. UI is never decorative.

Motion Principles

Framer-school motion: signals state change, never decorates. Developers have zero tolerance for gratuitous animation in data tools.

Hover — try it

transition: 150–180ms
easing: cubic-bezier(0.4,0,0.2,1)
PULSE (scanning only)
opacity 1 → 0.45 → 1
scale 1 → 0.7 → 1
1400ms ease-in-out infinite
PAGE ENTRY
opacity 0 → 1
translateY 12px → 0
300ms, 60ms stagger per card
FOCUS RING
box-shadow 0 0 0 3px
rgba(46,255,160, 0.10)
0ms — instant, no transition
NEVER: Animate layout shifts. Animate color alone without a trigger. Use spring physics on data tables. Animate more than 3 elements simultaneously. Add any animation that loops without user interaction.
13

Copy Voice

Sound like a senior engineer built the product. Not a marketing team. Not a chatbot. Every string should be reviewable by a developer.

✓ DO
"3 AWS services detected."
"Scan failed. GitHub returned 403."
"Connect GitHub →"
"Scanning 247 files…"
"Never scanned"
"New S3 bucket detected in push to main"
✗ DON'T
"We found 3 amazing AWS services!"
"Oops! Something went wrong. Please try again."
"Get Started with GitHub Integration"
"Please wait while we process your repository…"
"This repository has not been scanned yet"
"We detected a new S3 bucket in your recent push!"
Headlines
Verb first or strong noun. Max 8 words. No full stops. No exclamation marks.
Subtext
One sentence. Max 120 chars. Answers "what does this do?" in plain terms.
CTAs
Verb + object: "Scan repo" "View report" "Add alert" — never "Click here" or "Submit"
Errors
What happened + error code. Not why you're sorry. "GitHub returned 403 — check your OAuth token."
Empty states
One sentence + one action button. No walls of explanatory text. "No repos connected. Connect GitHub →"
14

Visual Do's & Don'ts

Guardrails derived from deep analysis of both reference sites. These apply to every screen in CloudLens.

✓ Visual Do's
Use --accent sparingly — it signals action
One subtle radial glow per hero section, nothing else
DM Mono for all technical values and repo names
Animate status transitions only (scanning → done)
Dark surface → darker surface nesting for depth
Borders to separate UI regions — never shadows
Compact tables with 1.65 line-height minimum
✗ Visual Don'ts
No light mode surfaces inside dark layouts
No purple gradients (cliché dark UI aesthetic)
No drop shadows on cards — use borders instead
No more than 2 accent colors visible per screen
No decorative icons without a semantic purpose
No text below 12px in any data surface
No color alone to distinguish states — always pair with shape/icon
